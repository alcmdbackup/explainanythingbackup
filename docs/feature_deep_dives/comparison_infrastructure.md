# Comparison Infrastructure

## Overview

The comparison infrastructure extends the evolution pipeline with a persistent **Article Bank** system for comparing text quality across different generation methods (1-shot LLM generation, evolution pipeline winners, and evolution baselines). It uses an Elo rating system with bias-mitigated pairwise comparisons to rank articles, and provides cost-efficiency metrics (`elo_per_dollar`) to answer the question: "Does spending more on generation produce better quality?"

The system operates through three workflows:
- **Workflow A** (1-shot): Generate an article from a prompt using any supported LLM model, add to bank
- **Workflow B** (evolution): Add an evolution pipeline winner (and optionally its baseline) to the bank
- **Workflow C** (comparison): Run pairwise comparisons between all entries in a topic using a judge LLM

## Key Concepts

### Article Bank

A persistent store organized by **topics** (unique prompts). Each topic contains **entries** â€” articles generated by different methods/models. Entries are ranked via Elo ratings computed from head-to-head LLM-judged comparisons.

### Bias-Mitigated Comparison

Every pairwise comparison runs twice with reversed presentation order (A-vs-B, then B-vs-A) to mitigate position bias. The standalone `compareWithBiasMitigation()` function (extracted from the evolution pipeline's CalibrationRanker) uses order-invariant SHA-256 cache keys for deduplication. Results include a confidence score (1.0 = both rounds agree, 0.5 = disagreement treated as low-confidence draw).

### Elo Rating System

- Initial rating: 1200
- K-factor: 32
- Confidence-weighted scoring: `scoreA = 0.5 + 0.5 * confidence` for winner A, blending toward 0.5 for uncertain results
- `elo_per_dollar = (elo_rating - 1200) / total_cost_usd` â€” negative values indicate the article underperforms the baseline Elo relative to its cost

### Multi-Provider LLM Support

Articles can be generated using OpenAI (gpt-4.1, gpt-4.1-mini, gpt-4o), DeepSeek (deepseek-chat), or Anthropic (claude-sonnet-4) models. The `callLLMModel` router dispatches to the correct provider based on model prefix. Comparisons use a separate judge model (typically `gpt-4.1-nano` for cost efficiency).

## Architecture

### Database Schema

Four tables (migration `20260201000001_article_bank.sql`):

| Table | Purpose |
|-------|---------|
| `article_bank_topics` | Prompt-based grouping with case-insensitive unique index on `LOWER(TRIM(prompt))` |
| `article_bank_entries` | Articles with FKs to evolution runs/variants, `generation_method` CHECK constraint, soft-delete via `deleted_at` |
| `article_bank_comparisons` | Head-to-head match history with confidence scores and judge model |
| `article_bank_elo` | Per-topic Elo ratings with `elo_per_dollar` cost-efficiency metric, unique constraint on `(topic_id, entry_id)` |

### Server Actions

All actions follow the `ActionResult<T>` + `requireAdmin()` + `withLogging` + `serverReadRequestId` pattern:

| Action | Purpose |
|--------|---------|
| `addToBankAction` | Select-or-insert topic (case-insensitive ilike match) + entry insert + Elo initialization |
| `generateAndAddToBankAction` | Generate article via LLM (title + content) then add to bank |
| `getBankTopicAction` | Single topic by ID |
| `getBankTopicsAction` | All topics with aggregated stats (entry count, Elo range, cost, best method) |
| `getBankEntriesAction` | All active entries for a topic |
| `getBankEntryDetailAction` | Full entry detail including metadata |
| `getBankLeaderboardAction` | Elo-ranked entries joined with entry details |
| `runBankComparisonAction` | All-pairs comparison with configurable rounds, updates Elo |
| `getCrossTopicSummaryAction` | Cross-topic aggregation by generation method |
| `getBankMatchHistoryAction` | All comparisons for a topic |
| `deleteBankEntryAction` | Soft-delete entry + cascade comparisons/Elo |
| `deleteBankTopicAction` | Soft-delete topic + cascade entries/comparisons/Elo |

### CLI Scripts

| Script | Purpose |
|--------|---------|
| `scripts/generate-article.ts` | Standalone article generation with `--bank` flag for auto-insertion |
| `scripts/run-bank-comparison.ts` | CLI for running pairwise comparisons with multi-provider judge support |
| `scripts/add-to-bank.ts` | Add existing evolution run winner to bank |
| `scripts/run-evolution-local.ts` | Extended with `--prompt` for seed generation and `--bank` for auto-insertion |

### Admin UI

Two pages under `/admin/quality/article-bank/`:

**Topic List Page** (`page.tsx`):
- Cross-topic cost efficiency summary cards (per generation method)
- Topics table with entry counts, Elo ranges, costs, best method badges
- "Generate New Article" button with topic picker dropdown (select existing topic or create new), model selector, generates via LLM, adds to bank
- "New Topic" button for empty topic creation

**Topic Detail Page** (`[topicId]/page.tsx`):
- 4-tab layout: Leaderboard, Cost vs Elo, Match History, Compare Text
- Elo leaderboard with expandable inline rows showing generation metadata
- Evolution entries link directly to run detail page (1-click debug path)
- Agent cost breakdown and meta-feedback display for evolution entries
- Cost vs Elo scatter chart (Recharts, SSR-disabled) with bidirectional linking
- Word-level text diff using `diffWordsWithSpace`
- "Run Comparison" dialog with judge model + rounds selector
- "Add from Evolution Run" dialog listing completed runs

**Integration with Evolution Run Detail**:
- "Add to Bank" button on completed run pages
- Dialog with prompt input, baseline checkbox, winner preview
- Success toast with link to navigate to the bank topic

## Data Flow

### Workflow A: 1-Shot Generation â†’ Bank
```
User enters prompt + selects model in UI
  â†’ generateAndAddToBankAction
    â†’ createTitlePrompt â†’ callLLMModel (title generation)
    â†’ createExplanationPrompt â†’ callLLMModel (article generation)
    â†’ Select-or-insert topic (case-insensitive ilike match)
    â†’ Insert entry (generation_method='oneshot')
    â†’ Initialize Elo (1200, match_count=0)
```

### Workflow B: Evolution Winner â†’ Bank
```
Admin clicks "Add to Bank" on completed run
  â†’ AddToBankDialog fetches variants, finds winner
  â†’ addToBankAction (generation_method='evolution_winner')
  â†’ Optional: addToBankAction for baseline (generation_method='evolution_baseline')
```

### Workflow C: Pairwise Comparison
```
Admin clicks "Run Comparison" on topic detail
  â†’ runBankComparisonAction(topicId, judgeModel, rounds)
    â†’ Fetch all active entries
    â†’ For each round:
      â†’ For each pair (i, j):
        â†’ compareWithBiasMitigation(textA, textB, callLLM)
        â†’ Record comparison (winner, confidence, judge)
        â†’ Update in-memory Elo (confidence-weighted)
    â†’ Persist updated Elo ratings to DB
```
