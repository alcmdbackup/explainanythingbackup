## High Level Summary
The tool is a new admin page under `/admin/evolution/*` that lets a researcher pick a rewrite agent family (generate-tactic or paragraph-recombine), edit the prompt text + settings (temperature, model, tactic/directive), supply a source article (or paragraph), then dispatch one or more configurations **in parallel** and view each model's raw output side by side, with per-config cost. It is a "playground" / dry-fire harness that invokes the rewrite step in isolation â€” NOT a full generateâ†’rankâ†’evolve run.

Key reuse opportunities (to confirm during planning):
- Prompt construction: `buildEvolutionPrompt()` / tactic templates in `evolution/src/lib/core/tactics/generateTactics.ts`; `FORMAT_RULES` in `evolution/src/lib/shared/formatRules.ts`; paragraph rewrite prompt in `evolution/src/lib/core/agents/paragraphRecombine/buildParagraphRewritePrompt.ts`.
- Single-rewrite invocation: `GenerateFromPreviousArticleAgent.execute()` (one variant per call) and the per-slot rewrite path inside `ParagraphRecombineAgent`. Local/mock paths exist in `evolution/scripts/run-evolution-local.ts` (`--mock`, `--prompt`).
- LLM client + cost: `createEvolutionLLMClient()` (`evolution/src/lib/pipeline/infra/createEvolutionLLMClient.ts`); pricing `src/config/llmPricing.ts`; model registry `src/config/modelRegistry.ts` (incl. per-model `maxTemperature`); cost via per-invocation `AgentCostScope.getOwnSpent()`.
- Temperature: `StrategyConfig.generationTemperature` (0â€“2, validated vs model `maxTemperature`); paragraph temperature ladder `paragraphRewriteTemperature(index, M, modelCap)`.

- Persist playground sessions/outputs to DB, or ephemeral in-memory only? (Affects schema + migrations.)
