## Summary (30 findings)
Surfaces swept: **Criteria** (list, detail tabs, New dialog, Edit dialog, RubricEditor) and **Tools** — Match Viewer (list filters, detail re-judge sandbox), Judge Rubrics (list, New/Edit inline forms, dimension picker), Judge Lab (launcher, leaderboard, test-sets list+detail+Edit/Clone, run detail, match history, agreement, pair-banks, escalation mode), Implied Rubric Weights (list, session Judge/Results tabs, new-session form, auto mode), Prompt Editor (config cards, presets, unit toggle).

Strongest functional bugs: **T21** (inferred-rubric Edit loads weights as 0–1 fractions → "1%/100%", Save permanently disabled), **T4** (RubricEditor setState-during-render on "+ Add anchor"), **T16** (Prompt Editor "+ Add config" labels skip: 1,3,5…), **T1** (raw Zod-issue JSON shown as the validation message), **T8** (test criteria leak into the rubric dimension picker), **T30** (re-judge rubric defaults to Article for paragraph matches), **T26** (criteria Metrics RUNS=40 vs Runs tab=12).
Systemic patterns: silent no-op on missing/invalid input (#23 / T17 / T20 / T24); no `<h1>` across Judge Lab subroutes (T9); toggle buttons lack `aria-pressed` (T23); dialogs miss `aria-describedby` (T3 ≡ #15); generic/inconsistent page titles (T7/T10/T11); tabs not URL-synced (T13); leaderboards/lists inconsistently sortable (T27 ≡ #6/#12).

> **Side-discovery during this sweep:** T8's investigation revealed my earlier #1/#9 fix migration (`20260623000001`) backfilled only 3 of the 7 tables carrying `is_test_content`. Fixed in the same migration — it now also re-flags `evolution_criteria`, `evolution_judge_rubrics`, `evolution_style_fingerprints`, `evolution_weight_inference_sessions`.
