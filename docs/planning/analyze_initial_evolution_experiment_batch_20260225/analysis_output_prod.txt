  8. Key Findings Summary
════════════════════════════════════════════════════════════

  1. deepseek-chat dominates: The only strategy with positive Elo/$ uses
     deepseek-chat for both generation and judging. All other model combinations
     produce below-baseline quality (Elo < 1500).

  2. Judge model matters most (Initial experiment): gpt-5-nano judge > gpt-5-mini
     judge by 63 Elo. Cheaper judge actually produced better results.

  3. Generation model matters most (Test experiment): claude-sonnet-4 gen > deepseek-chat
     gen by 86 Elo. But only 4/8 runs completed — partial data.

  4. High failure rate (50%): Mostly watchdog timeouts and batch runner incompatibility.
     Need to fix failure modes before scaling experiments.

  5. Budget too low for multi-round experiments: Both experiments exhausted budget
     in Round 1. At $0.10/run and 8 runs/round, need at least $2-3 for 2-3 rounds.

  6. No oneshot baseline in production HoF: Can't compare evolution vs oneshot quality.
     Need to run prompt bank comparisons to establish baselines.

  7. iterativeEditing is the workhorse: 150 avg Elo gain per run, 12 samples.
     Generation creates the pool, iterativeEditing refines it.

  8. More iterations help: The single 5-iteration run (Quality) dramatically
     outperformed all 3-iteration runs (29.8 vs 8.4-22.6 ordinal).
